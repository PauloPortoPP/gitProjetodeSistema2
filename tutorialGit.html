Projeto básico com Git e Github

Tutorial Git e Github

Resumidamente, os passos para o trabalho são:


1 Criar repositório no github chamado 'gitProjetodeSistema2';
2 Adicionar todos ao projeto. Selecione o projeto que quer compartilhar(no caso 'gitProjetodeSistema2') e depois vá em Settings/Collaborators para adicionar os nomes das pessoas irão compartilhar o projeto.
3 Caso não tenha configurado o git, siga os passos abaixo:
  • Configurar senha e usuário para o git: git config --global user.name seunome e git config --global user.email;
  • Gerar e cadastrar uma chave ssh. Para gerar vá no git e digite no terminal ssh-keygen -t rsa -C 'algo para identificar usuario'. Após informar uma passphrase, copie conteúdo do arquivo gerado (normalmente em .../.ssh/id_rsa.pub). No github, vá em Account Settings/SSH Keys, clique no botão Add SSH Key e cole o conteúdo copiado do arquivo id_rsa.pub;
4 No Windows Explorer, criar uma pasta chamada 'gitProjeto' no seu diretório home (no Windows c:\Users\seunome\)
5 No git, acessar a pasta recém criada através do comando cd gitProjeto.
6 Iniciar um git neste diretório com git init;
8 Adicionar repositório (do nosso exemplo) ao git através do comando git remote add origin https://github.com/PauloPortoPP/gitProjetodeSistema2.git
10 Crie um clone do projeto na sua máquina(na pasta gitProjeto). No terminal do git, digite git clone https://github.com/PauloPortoPP/gitProjetodeSistema2.git
11 Crie os arquivos definidos na pasta de acordo com as tarefas postadas no github na aba Issues
12 No terminal, adicione todas as alterações (os arquivos novos/modificados) ao seu repositório local git com git add .
13 Para confirmar estas mudanças, digitamos no terminal git commit -m "comentários das alterações" ;
14 Ainda no terminal, enviando para o repositório remoto (github): git push -u origin master. Caso obtenha um erro será necessário executar o comando git fetch origin e logo após o comando git push -u origin master;
